How it works

Three steps. The hard part — turning fun into data — is automatic.

01

Describe it

Type one sentence. A Director agent invents the round-based game; parallel Builder agents lay out the scene and rules — all on Gemma 4 / Cerebras at ~100 ms a call.

02

People play it

The game is genuinely fun — but winning requires good arm control: reach, grasp, transport, place. A vision Playtester screenshots the build and auto-fixes it until it grades well.

03

You harvest data

Every frame is recorded as a step — joint angles · action · reward · skill. Sessions stream out as a clean JSONL dataset you can export straight to LeRobot.

The real trick

Order the data distribution you need.
Gemma designs games that elicit it.

Short on insertion attempts? Need more cluttered-scene grasps or left-side approaches? Pick the skills — the Director shapes the rules so winning produces exactly those motions.

Skills to emphasize
grasp place sort stack insert transport

Selected skills become the optimization target. The next batch of games will be biased toward producing grasp + place trajectories — without ever telling players they're labeling data.
